\abstract

Design by Contracts is a development strategy focuses on placing contracts in
function arguments, ensuring inputs and outputs are valid during runtime.
Moreover, once a contract is broken, the runtime can report the developer which
part of the code is responsible. In this thesis, we attemp to implement contracts
using effect handlers, a unified approach to computational effects. We showed that
contracts can be implemented as a wrapper around effect handlers. We also provided
an implementation in OCaml, a language that has native support for effect handlers
but not for contracts.
